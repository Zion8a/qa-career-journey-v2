# QA Career Journey

Detta repo dokumenterar min resa mot en modern och framtidssäker roll som mjukvarutestare. Fokus ligger på att bygga både praktisk kompetens och konkreta leverabler som visar hur jag arbetar med kvalitet, testdesign, buggrapportering, API-testning, UI-testautomation och CI.

## Syfte
Målet med detta repo är att steg för steg bygga en portfolio som visar:
- hur jag tänker som testare
- hur jag dokumenterar testarbete professionellt
- hur jag arbetar med både API- och UI-testning
- hur jag bygger automation och kopplar testning till CI
- hur jag utvecklar en profil som matchar framtidens QA-roll

## Kompetensområden i repot
Detta repo visar arbete inom:
- kvalitetstänk och riskförståelse
- testdesign
- buggrapportering
- explorativ testning
- API-testning med Postman
- CLI-körning med Newman
- GitHub Actions
- UI-testautomation med Playwright
- selectors, assertions och användarflöden
- Page Object Model
- negativa scenarier och validering
- CI och professionell paketering

## Struktur
### Part 01–06: QA-grund, API och CI
- `part-01-foundation` – grund i kvalitetstänk, risk och testperspektiv
- `part-02-test-design` – krav, testfall, gränsvärdesanalys, ekvivalensindelning och beslutstabell
- `part-03-bug-reporting` – buggrapporter och exploratory session report
- `part-04-api-testing` – API-testning i Postman
- `part-05-newman` – lokal körning av API-tester med Newman
- `part-06-github-actions` – GitHub Actions-workflow för API-testning

### Part 07–12: Playwright och modern UI-testautomation
- `part-07-playwright-foundation` – första Playwright-projektet och första UI-smoketestet
- `part-08-playwright-elements-and-assertions` – robust elementidentifiering och tydliga assertions
- `part-09-playwright-form-and-user-flow` – användarinteraktion med klick, input och resultatverifiering
- `part-10-playwright-page-object-model` – strukturering av testkod med Page Object Model
- `part-11-playwright-negative-tests-and-validation` – negativa scenarier, validering och felhantering
- `part-12-playwright-ci-and-professional-packaging` – GitHub Actions för Playwright samt professionell paketering

## Varför detta är relevant för framtidens QA-roll
Rollen som mjukvarutestare utvecklas mot att bli mer teknisk, mer analytisk och mer integrerad i utvecklingsflödet. Det här repo:t är byggt för att spegla den utvecklingen, med fokus på:
- kvalitetstänk och risk
- teknisk testning av API och UI
- automation
- CI
- tydlig dokumentation
- professionell paketering av testarbete
