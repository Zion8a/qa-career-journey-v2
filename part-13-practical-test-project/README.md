# Part 13 – Practical Test Project

This section documents my work during the course Practical Project Work in Software Testing as part of my education as a Software Tester at EC Utbildning.

The project is based on manual testing of Blekingetrafiken's web-based travel planner.

```text
https://blekingetrafiken.se/reseinformation/sok-resa/
```

The purpose of this project is to show how I work with structured testing in a realistic project context. The focus is on test planning, exploratory testing, test case design, bug reporting, regression testing, requirement review and communication of test results.

## Project objective

The main test objective is to identify serious problems in Blekingetrafiken's online travel planner that may affect users when searching for, planning and understanding a journey.

## Course focus

This part connects to the following areas:

- Software development processes and testing
- Static testing
- Testing in agile teams
- Test techniques
- The test process
- Test reporting and communication
- Critical review of requirements

## Tools and methods

The project work uses:

- Reqtest for test suites, test cases, test execution and bug reporting
- Agile Board for planning and follow-up
- Area tour for exploratory analysis
- Manual functional testing
- Risk-based prioritization
- Regression test selection
- Structured test documentation
